Improved Functions   

  • The 2D viscous layer design functions MakeEdgeOffset, ExtendViscousLayer and CloseViscousLayer are now able to return faces by setting dim = 2.
  • The ExtendSpline function allows now to force the extension to be straight from the input edge end point to the target position. The user can manage this feature through the strat argument, which can equal "flex" or "rigid".
  • The SplitEdge and RebuildSpline functions allow now to give a list of parameters instead of a number of points.
  • In the FuseSplines function, the number of points given in the np argument is now divided up between edges to fuse according to their length. This allows to fuse without problem splines having a very high length difference.
  • Many functions are now taking into account the curvature of input edges to give more precise results while minimizing the necessary number of internal points used to rebuild or create shapes. This new feature can be disabled through a new Boolean argument curv which was added to the following functions:
     
    • FuseSplines
    • ExtendSpline
    • FuseSplineSets
    • MakeEdgeOffset
    • MakeTipViscousLayer
  • Many functions can now return shapes separately, so as to allow the user to get them in the order found by the cfdmsh function instead of getting a single compound. The return type can be managed through a new Boolean argument single which was added to the following functions:
     
    • RebuildSpline
    • SplitEdge
    • FuseSplines
    • ExtendSpline
    • RebuildFace
    • MakeMiddleSpline
    • MakeCurveFromUnsortedVertexes
    • MakeEllipticalFilling
    • MakeFillingFromUnsortedEdges
    • MakeEdgeOffset
  • In the following functions, the default name of exported/imported files was modified so as to avoid them to replace by mistake an existing file not linked with cfdmsh.
     
    • ExportGeometricalGroups
    • ImportGeometricalGroups
    • ExportMeshConfiguration
    • ImportMeshConfiguration
    • ExportHypotheses
    • ImportHypotheses

Other Modifications   

  • Since cfdmsh is before all optimized for GUI use, the order of arguments could change a bit due to the decrease of the importance of the np argument in functions taking now into account the input edge curvature.
