Import   

Once installed, the cfdmsh library can be used from the Salome GUI or in any Python script.

In both cases, cfdmsh has to be loaded ("imported") before to use it, each time a new Salome session is opened. The way to proceed depends on the way it was installed.

Method 1 - a   

If the method 1 was followed during the installation, the user can type directly in the Python console of Salome or in a Python script:

import cfdmsh

and then call any cfdmsh function by typing: cfdmsh.<function name>(<list of arguments>). For example:

cfdmsh.RebuildSpline( 30, 'Edge_1' )

In this case, it is also possible to have a quick help on a cfdmsh function by typing: help(cfdmsh.<function name>). For example:

help( cfdmsh.RebuildSpline )

Method 1 - b   

Always after installing cfdmsh using the method 1, the user can also load it typing:

from cfdmsh import *

In this case, the user has not anymore to type cfdmsh. before calling a cfdmsh function. For example:

RebuildSpline( 30, 'Edge_1' )

And, to have a quick help on a cfdmsh function, the user can type: help(<function name>). For example:

help( RebuildSpline )

Method 2   

If the method 2 was followed during the installation, the cfdmsh.py file has to be loaded using File > Load Script or [Ctrl]+[T] or by typing in a Python script: execfile(r"<cfdmsh.py_path>"), knowing the path is taken from the user's home folder when the command is executed from the GUI of Salome. For example:

execfile( r"/home/user/cfdmsh/cfdmsh.py" )

or:

execfile( r"cfdmsh/cfdmsh.py" )

Then, the way to call a cfdmsh function is the same as for the method 1-b. That is, by typing: <function name>(<list of arguments>). For example:

RebuildSpline( 30, 'Edge_1' )

And, the command to have a quick help on a cfdmsh function is also the same as for the method 1-b. For example:

help( RebuildSpline )

Function List   

Once imported, the list of all implemented cfdmsh functions can be displayed in the Python console typing:

PrintDefinedFunctions( )

or, simply:

pdf( )

To have a condensed list in alphabetical order, the user can type:

pdf( 1 )

Selection by Name   

In a Python script

Almost all cfdmsh functions need to be executed using existing objects created in Salome as input arguments (eg. edges, faces, meshes, etc.).

When cfdmsh is used inside a script, the user can indicate these objects using their name in the script. For example, to fuse two splines Edge_1 and Edge_2, the user can type:

fs( [Edge_1, Edge_2] )

From the GUI

Now, if the user is using cfdmsh from the Salome GUI, he can designate them through their name in the study tree. For example, to fuse two splines Edge_1 and Edge_2, the user can type:

fs( ['Edge_1', 'Edge_2'] )

Notice that the only one difference with the previous example is the presence of brackets around Edge_1 and Edge_2.

GUI selection   

For some functions, input shapes can even be selected in the Salome GUI without giving their name in the Python console.

For example:

RebuildSpline( 30 )

will give the same result as:

RebuildSpline( 30, 'Edge_1' )

if Edge_1 is previously selected in the Salome GUI (eg. by left-clicking on it in the study tree or in the 3D window).

Recursive Functions   

Also, some cfdmsh functions, called "recursive", allow their main input shape argument to be an list instead of a single value. For example:

RebuildSpline( 30, ['Edge_1', 'Edge_2', 'Edge_3'] )

will give the same result as:

RebuildSpline( 30, 'Edge_1' )
RebuildSpline( 30, 'Edge_2' )
RebuildSpline( 30, 'Edge_3' )

as well as:

RebuildSpline( 30 )

if Edge_1, Edge_2 and Edge_3 are previously selected in the Salome GUI.

Acronyms   

To accelerate the cfdmsh use, all functions can be called using either their full name or their acronym in lower case. For example:

pvli( 'Compound_1' )

will give the same result as:

PropagateViscousLayerIntersection( 'Compound_1' )

Auto-completion   

Finally, from the version 7.2.0 of Salome, auto-completion is available in its Python console. The cfdmsh user is strongly encouraged to take advantage of this very useful tool.

To have an example of auto-completion, the user can load the cfdmsh.py file using [Ctrl]+[T] and type:

Reb

Then, using the [Tab] key should automatically replace Reb with Rebuild and display below in green the different functions beginning with Rebuild.

Typing F and using the [Tab] key should replace the line with RebuildFace.

As visible here, auto-completion allows to both accelerate the Python console use and to quickly see available functions.
