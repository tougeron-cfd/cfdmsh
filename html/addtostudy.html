AddToStudy

Short Description   

Flexibly publishes an object in the study tree.

Additional Information   

This function tries to make the shape publication as simple as possible.

Inputs can be single shapes or lists of shapes. Giving them a name for publication is mandatory. In addition, the name argument can be something else than a string (eg. an integer or float). Also, the add of a suffix (eg. "_1") at the end of the publication name is automatic.

The publication is followed by the automatic display of the shape as well as the automatic refresh of the study tree.

Each of automatic features can be separately disabled so as to speed up the publication of huge numbers of shapes.

Definition   

def AddToStudy(object, name = "Geometrical Object", father = None, suffix = True, disp = True, refresh = True):

Argument(s)   

Name Description Type GUI selection [?] Selection by name [?] Recursive [?] Default value
object The object to publish. Any geometrical object - - yes -
name The name to give to the shape in the study tree. String - - - "Geometrical Object"
father The geometrical shape in which to add the published shape. Any geometrical object - - - None
suffix Determines if a suffix (eg. "_1") as to be added to the name of the shape to publish. Boolean - - - True
disp Determines if the shape has to be displayed in the 3D window after publication. Boolean - - - True
refresh Determines if the study tree has to be refreshed after publication. Boolean - - - True

Returned Value(s)   

dim Value [?] single Value [?] Type Number Name
- - - - -

Example   

from cfdmsh import * # To adapt to the cfdmsh installation method

v_1 = geompy.MakeVertex(0, 0, 0)
v_2 = geompy.MakeVertex(0, 1, 0)
v_3 = geompy.MakeVertex(0, 0, 1)

AddToStudy([v_1, v_2])
AddToStudy(v_3, "Vertex")
