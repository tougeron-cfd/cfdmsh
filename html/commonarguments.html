Common Arguments

Here are listed the arguments which are often used in cfdmsh.

add   

The add argument defines if the objects which are returned by the function have also to be added to the Salome study. It should be set to True if the returned objects have to be added or False if the returned objects have not to be added.

curv   

The curv argument defines if curvature of the input edges has to be taken into account or not. If equals False, input shapes are split into the number of points defined by the np argument. If equals True, additional points are added according to the local curvature of the input edge.

dim   

The dim argument defines the desired dimension of the objects to be returned. It should be set to 0 if Vertexes have to be returned, 1 if Edge have to be returned, 2 if Faces have to be returned or 3 if Solids have to be returned. A negative value of this argument means that the function won't return any geometrical shape.

dist   

The dist argument is commonly a distance from the input geometrical object to be used during the function execution (offset distance, extension distance, etc.).

face   

The face argument is a Face (or its name in the study tree) of which the normal is used to define offset or extension direction.

file   

The file argument defines the name of the file which is read / writen during the function execution.

infa   

The infa argument defines if the shapes are added to the study tree inside their main father object or not.

np   

The np argument defines the number of points used during the rebuild or construction of internal or returned object (during spline cutting, offset construction, etc.). See also the curv argument.

plane   

The plane argument is a Face (or its name in the study tree) defining a plane in which are built the intermediate edges created during the function execution.

rev   

The rev argument reverses the value of the dist or thick argument when it equals True (else False).

style   

The style argument defines if edges created during the function execution have to be straigth or bended. Its value can be "straight" or "smooth".

single   

The single argument defines if the returned shapes have to be returned separately or in a single compound or shell.

tol   

The tol argument is commonly a distance under which two points are supposed to be coincident. It is commonly used during sorting operations.
