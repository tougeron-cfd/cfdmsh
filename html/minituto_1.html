Mini-Tutorial 1

The purpose of this tutorial is to show how to manage geometrical groups using cfdmsh.

Cfdmsh allows to copy groups between shapes according to their geometrical description or according to their index inside the shape.

The following cfdmsh functions will be used:

  • PutAllSubShapesInAGroup
  • CopyGeometricalGroups
  • ExportGeometricalGroups
  • ImportGeometricalGroups

NB: This tutorial was tested on Salome 7.7.1 on Linux with the following settings:

File > Preferences... > Geometry > Default display mode > Wireframe.

File > Preferences... > Geometry > Hide input objects from viewer > OFF.

Content of this mini-tutorial:

  • Open a New Study
  • Create the Source Cube
  • Create Groups
  • Create the Target Cube
  • Copy Groups
  • Import/Export Groups

Open a New Study   

  1. Open Salome.
  2. Open a new study.

    3. File > New

  3. Load the Geometry module.

    4. Click on in the toolbar

  4. Expand the Geometry node in the study tree.

  5. Load the cfdmsh library.

    6. [Ctrl] + [T] → Select the "cfdmsh.py" file on your computer

Create the Source Cube   

  1. Create a cube.

    2. New Entity > Primitives > Box → Apply and Close

  2. Hide the trihedron.

    3. Click on in the 3D window toolbar

Create Groups   

  1. Create three groups containing respectively all sub-vertexes, all sub-edges, and all sub-faces of the cube.

    2. Select "Box_1" → In the Python console, type:

    PutAllSubShapesInAGroup( 0 )
    PutAllSubShapesInAGroup( 1 )
    PutAllSubShapesInAGroup( 2 )

    or:

    passiag( 0 )
    passiag( 1 )
    passiag( 2 )

  2. Expand the "Box_1" node in the study tree to see newly created groups.

  3. Show only these groups.

    4. Select "Box_1" → Right-click > Show Only Children

Create the Target Cube   

  1. Translate the cube 200m following the X axis.

    2. Select "Box_1" → Operations > Transformation > Translation →

    Set "Dx" to 200 → Apply and Close

Copy Groups   

  1. Copy the groups from the first cube to the other.

    2. In the Python console, type:

    CopyGeometricalGroups( 'Box_1', 'Translation_1' )

    or:

    cgg( 'Box_1', 'Translation_1' )

  2. Expand the "Translation_1" node in the study tree to see copied groups.

  3. Show only these groups.

    4. Select "Translation_1" → Right-click > Show Only Children

    Only sub-shapes from "Box_1" having a sub-shape sharing the same topology in "Translation_1" were copied.

Import/Export Groups   

  1. Hide everything in the 3D window

    2. Right-click in the 3D window > Hide All

  2. Export groups from the source cube into a file.

    3. Select "Box_1" → In the Python console, type:

    ExportGeometricalGroups( )

    or:

    egg( )

    This command created a file called "cfdmsh_grps" in your home folder.

  3. Import exported groups into the target cube.

    4. Select "Translation_1" → In the Python console, type:

    ImportGeometricalGroups( )

    or:

    igg( )

    This time, the whole groups were copied from "Box_1" to "Translation_1", regardless the change in form or position of their sub-shapes. This was possible because both source and target geometries shared the same sub-shape ID organization.

    You can close this study.

END OF THIS TUTORIAL
